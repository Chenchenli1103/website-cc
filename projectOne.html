Background information

The project: This is a small modification of a long-term project in my full time work in a start-up. The product is an educational tool for Computer Science, and this modification is focused on course arrangement calendar. Our product manager had decided the functions and basic interface but wasn’t satisfied with it. My job was to make this page more elegant and efficient.

1. Testing for old version

We invited 5 colleagues from other departments to value our first version interface via heuristic evaluation . After that we collected some suggestions and confusions as further focus.

2. evaluation and problem defining

Constraints

1. Manpower is limited. We can’t make big change for this page which would cause burden to developer.

2. In order to save time, we can’t break fundamental framework of our interface.

3. The color of blocks are selected by users. It’s difficult to control color matching.

problems

1. Information and action icons need more space.

2. Action icons repeat in every color block and make this page too crowded. Moreover, they are too small to be clicked.

3. If the instructor arranges many courses and colors, the page would look dazzling and messy.

design goal

After taking both problems and constraints into account, my design goal became: to find a way to save space for information and show action icons more efficiently.

3. ideation and prototype

OLD VERSION

NEW SOLUTION

For limitations there was no possibility to expand block size. I thought of menus to hide action icons and quickly combined it with Motion Design. As is shown on the right, when mouse moves over, the menu falls down. When mouse moves out, it withdraws.

4&5 testing & visual design

Our product manager and I completed the testing part and slightly changed the time of motion.

In visual part I redefined the Color Library for instructors to select and did some small modifications to layout.
